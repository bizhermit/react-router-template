# アーキテクチャ

## 目的

本ディレクトリのドキュメントは、実装時の判断基準（責務・配置・記述ルール）を統一することを目的とする。

## 読む順番

1. [アプリケーション構成](./app.md)
2. [ディレクトリ構成](./directory.md)
3. [コーディング規約](./code.md)

この順番で読むことで、責務境界 -> 配置判断 -> 実装ルールの流れで理解できる。

変更時のレビュー観点として、[セキュリティレビュー指針](./security-review.md)も参照する。

## 技術スタック

- Node.js v24
- TypeScript v5
- React v19
- ReactRouter v7
- BetterAuth v1
- TailwindCSS v4
- PostgreSQL v18

## アプリケーション構成

責務境界・依存方向・処理フローを定義する。

詳細は[アプリケーション構成](./app.md)を参照。

## ディレクトリ構成

実装をどこへ配置するかの判断ルールを定義する。

詳細は[ディレクトリ構成](./directory.md)を参照。

## コーディング規約

実装時の記述ルールとレビュー観点を定義する。

詳細は[コーディング規約](./code.md)を参照。

## セキュリティレビュー指針

変更内容に応じて確認すべきセキュリティ観点と、チェックリストへの導線を定義する。

詳細は[セキュリティレビュー指針](./security-review.md)を参照。
